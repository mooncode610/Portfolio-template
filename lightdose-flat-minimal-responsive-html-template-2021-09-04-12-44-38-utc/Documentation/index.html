Introduction

Light Dose is a Flat & Minimal One Page Responsive Template. It was created to be simple-looking, yet powerful and flexible enough to fit your personal needs or match your corporate style. We offer you an up-to-date and really high quality work with outstanding design and well-thought usability approach.

Please do not forget to rate «Light Dose» on ThemeForest. This can be done in Downloads area. Just select «Light Dose» in the Download list and rate it. We really appreciate your feedback and will be grateful for your opinion.

Besides, we will be happy to meet you in social networks and on websites.

Getting Started

This guide is targeted at people, who have at least basic knowledge of html markup language. Furthermore css and javascript knowledge is advised. If you do not feel yourself well with understanding these technologies you should consider finding someome to assist you.

This theme was designed using Twitter Bootstrap 3.0. A link to documentation is provided in External reference section of this guide.

A jQuery javascript library with various plugins is used. A link to documentation is provided in External reference section of this guide.

To install this template into a CMS of your choice you should consult it's user manual.

To cover some more details, here is the list of common questions that are often asked:

  1. I want to customize an X feature. How can I do it?

    Is it a part of this template? Then it in most cases it can be customized via changing of html structure without any intrusion to other files.

    Is it a plugin, code snippet or module, used in this template? Then in the first place you must refer to that plugin manual page. This guide doest not fully cover third party plugin configuration.

  2. I have X content blocks in a row, but I want Y instead!

    Twitter Bootstrap 3.0 allows you to split a block into separate by using its grid system. It has also a lot of other useful features, such as its own icon set and other components. A link to documentation is provided in External reference section of this guide.

    For a extra-short explanation one can see a Class reference section of this guide.

Preloader

Each page of this template contains a preloader code by default. It will be disabled upon a removal of this code. 

							<!--BEGIN Preloader-->
							<!--DESCRIPTION

								Spinner configuration can be changed using:
									http://fgnass.github.io/spin.js/
								After configuration <Spinner options> section should be replaced with desired options.
							-->
							<script type="text/javascript" src="js/spin.min.js" charset="utf-8"></script>
							<script type="text/javascript">
								var element = document.createElement( 'div' );
								element.id = 'spin';
								element.style.position = 'fixed';
								element.style.zIndex = 10000;
								element.style.top = 0;
								element.style.right = 0;
								element.style.bottom = 0;
								element.style.left = 0;
								element.style.width = '100%';
								element.style.height = '100%';
								element.style.background = '#FFF';
								document.body.appendChild( element );
								//	Starting a spin

								//<!--BEGIN Spinner options-->
								var opts = {
								  lines: 7, // The number of lines to draw
								  length: 0, // The length of each line
								  width: 7, // The line thickness
								  radius: 7, // The radius of the inner circle
								  corners: 0.9, // Corner roundness (0..1)
								  rotate: 0, // The rotation offset
								  direction: 1, // 1: clockwise, -1: counterclockwise
								  color: '#000', // #rgb or #rrggbb or array of colors
								  speed: 1.2, // Rounds per second
								  trail: 85, // Afterglow percentage
								  shadow: false, // Whether to render a shadow
								  hwaccel: false, // Whether to use hardware acceleration
								  className: 'spinner', // The CSS class to assign to the spinner
								  zIndex: 2e9, // The z-index (defaults to 2000000000)
								  top: 'auto', // Top position relative to parent in px
								  left: 'auto' // Left position relative to parent in px
								};
								//<!--END Spinner options-->
								var target = document.getElementById( 'spin' );
								var spinner = new Spinner( opts ).spin( target );
							</script>
							<!--END	Preloader-->

To customize a spinner one you should look at spin.js in an External reference section of this guide.

«Slider»

Slider consists of short title (1), long title (2), used-defined content (3) and slider pagination (4). It's behaviour and content is controlled via html structure and some extra classes.

Slider behaviour: if next slide is not an image or a video (doesn't contain any visual content), then only it's data-short and data-full attributes are used to change the text.

An example of an image slide. 

							<img class="slide" src="img/slider/1.jpg" data-short="Light Dose" data-full="No Line on the Horizon" alt="No Line on the Horizon" />

An example of a text slide. 

							<div class="slide" data-short="Light Dose" data-full="Enjoy a Mexico tour">
							</div>

An example of a video slide. 

							<video class="slide" muted="muted" loop="loop" preload="" autoplay="autoplay" data-short="Light Dose" data-full="Flat & Minimal One Page Responsive Template">
								<source src="video.mp4" type="video/mp4">
								<source src="video.webm" type="video/webm">
								<source src="video.ogv" type="video/ogg">
								<img class="fallback" src="img/slider/1.jpg" alt="video" />
							</video>

data-short and data-full only change text when it differs from the one is currently displayed.

Please note, that html5 video must containt three various sources in different formats and a fallback image in order to be accesible in various browsers. Additionally, a fallback image automatically replaces video on mobile devices and tablets in order to improve performance and usability.

«Services»

This block uses multiple-layered background with a parallax effect. Parallax is applied by a .aparallax class name on the element with a background image. It consists of two layers, one is shown only when animation begins. And another is stacked on top of it. During the animation process it slowly fades in, and after that the content blocks pop out.

For a better understainding of how to customize this block here is the example. 

						<article class="services page-light-dose theme-color background foreground aparallax" id="services">
							<div class="underlay">
								<div class="overlay aparallax">
									<!--Header and container with content blocks go here-->
								</div>
							</div>
						</article>

After creating an html structure we assign two background images in a custom css stylesheet. The second one is the one to appear, and the first - is the one to hide. When first image is sharp and second is blurred, a perfect illusion of actual n-time blurring is created. 

							.services {
								background-image : url( '../img/background-services.jpg' );
							}
							.services .overlay {
								background-image : url( '../img/background-services-blurred.jpg' );
							}

«Works»

Please keep in mind that this block does not use a bootstrap grid.

This block is designed to hold a lot of items with a preview image, its title and a small remarque. Additionaly any classes can be specified to any item to be used as tags to filter the content of grid. The following two screenshots illustrate a basic filter behaviour. When a page is first loaded, all items are visible. Upon selecting a tag, items, that do not have it, disappear. This mechanism allows arranging items into groups in any desired way. Moreover, several classes, separated by comma, can be specified for filter at once, e.g. .classOne, .classTwo, .classThree and so on. Filters itself should be placed above the grid, see (1) on the following screenshot.

From inside the code it looks like this. 

							<div class="row tags text-center grid-filter">
								<a class="active" href="#" data-filter="*">
									All
								</a>
								<a href="#" data-filter=".custom">
									Custom
								</a>

So there are two filters specified. One will allow to display all items, while another - only those, that have a .custom class.

Now we create two items in a grid and add a .custom class to one of them. So upon clicking «Custom» filter only one item will be displayed. But when «All» filter is selected - everything is visible again.

Blur effect

This feature allows you to blur background images "on the fly". It can be used for:

  1. Blurring page background image.
  2. Blurring article background.
  3. Blurring background, which uses parallax effect.

To apply this effect you must add a .blur class or a blur="X" attribute, where X is a desired effect strength. Adding a class will mean, that a default value of 10 will be used. 

							<article class="pricing page-light-dose theme-color background foreground links aparallax" blur="10" id="pricing">
								…
							</article>

Class reference

Twitter Bootstrap 3.0

Bootstrap includes a responsive, mobile first fluid grid system that appropriately scales up to 12 columns as the device or viewport size increases. It includes predefined classes for easy layout options, as well as powerful mixins for generating more semantic layouts.

getbootstrap.com

Grid systems are used for creating page layouts through a series of rows and columns that house your content. Here's how the Bootstrap grid system works:

  • Rows must be placed within a .container for proper alignment and padding.
  • Use rows to create horizontal groups of columns.
  • Content should be placed within columns, and only columns may be immediate children of rows.
  • Predefined grid classes like .row and .col-xs-4 are available for quickly making grid layouts. LESS mixins can also be used for more semantic layouts.
  • Columns create gutters (gaps between column content) via padding. That padding is offset in rows for the first and last column via negative margin on .rows.
  • Grid columns are created by specifying the number of twelve available columns you wish to span. For example, three equal columns would use three .col-xs-4.

Basically all grid classes are split into four categories, based on a device width:

xs - Extra small devices Phones (<768px)
sm - Small devices Tablets (≥768px)
md - Medium devices Desktops (≥992px)
lg - Large devices Desktops (≥1200px)
